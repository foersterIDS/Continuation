corrector and stepsize control

'corrector' — determine corrector-method

'sphere' (default) | 'orthogonal' | 'ellipsoid' | 'ellipsoid2' | 'unique'
TODO: What does this mean?
Determines corrector method to find new solution.
  • 'sphere': Find new solution on a sphere with radius around current solution.
  • 'orthogonal': Find new solution on a line perpedicular to the tangent going through the current solution with length
  • 'ellipsoid': Find new solution on a ellipsoid.
  • 'ellipsoid2': Find new solution on a ellipsoid.
  • 'unqiue': Find new solution by simply increasing λ from start to end.

closed curve detection

'closed_counter' — counter for closed_curve_detection

3 (default) | scalar, integer > 0
This counter is used for the detection of a closed curve. It specifiies how many times the current point must show up in the list of already found points.

'closed_curve_detection' — Determine a closed curve

'off' (default) | 'on'
If activated it is investigated in continuation whether the explored curve is a closed one. If this is true, continuation breaks after closed_counter found points. A simple closed curve i.e. is a circle.

closed curve example

display

'display' — display progress

'on' (default) | 'off'
TODO: What is displayed?
This option controls the output in the console. If activated, at first the initial solution is shown:
If no initial solution is found, it's shown in the consol. The output looks like this whenever a next solution is found:
The following is shown:
  • at which λ the next solution is found
  • the next arc-length to use, based on corrector method
  • the loop counter: increased in every run of the loop (even if no valid solution was found)
  • the step counter: only increased if a valid solution is found in a loop run
  • how many iterations were need to find a solution / optimal iteration steps
If no valid solution is found, it's shown in the output. If homotopy is used, the console shows the start of the use and whether it worked or not.
Finally the elapsed time is shown in seconds. Moreover, errors and warnings plus detailed explainations are shown if problems occur.

plot

'plot' — Choose plot method

'basic' (default) | 'detail' | 'semilogx' | 'semilogy' | 'loglog' |'off'
  • 'basic': creates a figure showing selected variables over lambda from l_start to l_end
  • 'detail': creates a figure showing selected variables over lambda from l_start to l_end (upper left), the needed iterations per loop over the loop counter (upper right), an interesting variable (bottom left; this variable may either be selected by plot_var_of_interest or is the most changing one; this subplot also shows predictor and corrector) and the arc length over the loop counter (bottom right)
  • 'semilogx': similar to 'basic', but x-axis is logarithmic. Please do only use positive lambda values
  • 'semilogy': similar to 'basic', but y-axis is logarithmic. Please do only use positive variable values or consider using the abs()-function
  • 'loglog': similar to 'basic', but x- and y-axis are logarithmic. Please do only use positive values for lamdba and the variables or consider using the abs()-function.

'plot_pause' — Pause live plot until enter is pressed

false (default) | true | integer > 0
Use this option to stop the live plot until you press enter after clicking in command window. Choosing true stops the live plot after the initial solution is plotted. You can also pass an integer. In this case the plot stops after plot_pause steps. Consider the latter option when searching for errors. May be combined with 'plot', 'detail' to view the predictor and corrector. If you want to leave continuation press ctrl+c after clicking in command window.

'live_plot_fig' — Choose a figure to plot in

NaN (default) | scalar, integer > 0
You can chose a certain figure to plot in. If NaN is passed, a new figure is created. Do not use, when 'bifurcation' method 'trace' is activated.

'plot_var_of_interest' — Choose interesting variable

NaN (default) | integer > 0 (if 'plot_vars_index' is not on default, you must choose a variable of those selected in this option!)
Choose a variable of interest you'd like to see in the detail bottom left subplot. Choosing another plot option, this argument has no effect.

'plot_vars_index' — Choose variables to plot

1:length(var0) (default) | array of integers > 0 with unique elements and max length of length(var0)
By passing this argument you can explicitly say which variables should be plotted. If i.e. set to [1,4,5] the variables and are plotted.

Plotting example

plotting_method = "detail" % choose plotting method
plotting_vars_index = 1:3%choose variables to plot
if strcmp(plotting_method, 'detail')
plot_var_of_interest = 2

plot_var_of_interest = NaN;
end
else
plot_var_of_interest = NaN;
end
plot_pause = true
addpath('..\..\examples\test_cases')
addpath('..\..\examples\test_cases\testfun02_aux')
testfun02;
addpath('..\..\src')
if isempty(plotting_vars_index)
continuation(fun,v0,lams,lame,ds0,'bifurcation','mark','ds_max',ds_max,'display','off','plot',plotting_method);
else
continuation(fun,v0,lams,lame,ds0,'bifurcation','mark','ds_max',ds_max,'display','off','plot',plotting_method, 'plot_vars_index', plotting_vars_index,...
'plot_var_of_interest', plot_var_of_interest, 'plot_pause', plot_pause);
end
